# Prowl v0.1 — Demo & Acceptance Walkthrough

Manual + automated acceptance for the v0.1 vertical slice (`/prowl search <topic>`).
Companion to `v0.1-implementation-roadmap.md` Phase 8 (Tests + end-to-end demo).

## 0. Prereqs

- Docker stack up: SearXNG `:8888` and Firecrawl `:3002` (see `docker-compose.yml`).
- Model endpoint configured in `.env` (see §3).
- `bun` installed (workspace already linked).

## 1. Automated tests

```bash
# Type-check repo-wide (acceptance: passes)
bunx tsc --noEmit -p packages/core/tsconfig.json
bunx tsc --noEmit -p packages/pi/tsconfig.json

# Unit tests — core composer (mock ports) in both packages
cd packages/core && bun test
cd ../pi       && bun test
```

- **`packages/core/test/search.test.ts`** — mock-port tests for the `search` composer:
  - primitive order `PLAN → SCATTER → SYNTHESIZE → PRESENT`,
  - output shape (summary + sources, `Sources:` rendered),
  - **zero-scrape default** vs **bounded `--read` extraction** (Phase 7),
  - resilient extraction (snippet-only fallback on scrape failure),
  - selection/ranking policy bounds (`selectForExtraction` ≤ budget & ≤1 host & ≤2 engine;
    `selectForSynthesis` ≤ limit & ≤2 per host).
- **`packages/pi/test/smoke.test.ts`** — live smoke tests against the Docker stack:
  - `searxngClient.search` returns ≥1 result,
  - `scrapePort.scrape` returns non-empty markdown for a selected URL,
  - `modelClient.generate` returns non-empty text **when `PROWL_MODEL_API_KEY` is set**
    (skips cleanly otherwise).

## 2. Manual demo — default (snippets-only)

No Firecrawl is touched. SearXNG snippets → Qwen synthesis → rendered Pi output.

```
/prowl search "magnesium orotate personal experiences"
```

Expected:
- A synthesized 3–6 paragraph summary biased toward personal/archival sources.
- A `Sources:` list with 3–5 source URLs.
- **Zero** Firecrawl calls (snippets-only default).

## 3. Manual demo — `--read` evidence mode

Bounded extraction: a diverse 3–5 URL set is scraped and its markdown flows into
synthesis. Firecrawl is invoked **only** for that bounded set.

```
/prowl search --read "magnesium orotate personal experiences"
```

Expected:
- Same summary shape, but the cited sources carry extracted Markdown evidence.
- Exactly `min(3–5, gathered)` URLs scraped (never every result).
- All other results use snippets.

## 4. Model configuration (for the live model smoke test + real demos)

`.env` (or shell) must define the OpenAI-compatible endpoint:

```dotenv
# ── Prowl model — single OpenAI-compatible endpoint (Qwen) ──────────────────
PROWL_MODEL_API_KEY=sk-...            # required for live model + demos
PROWL_MODEL_BASE_URL=https://<gateway>/v1   # Venice / OpenRouter / local proxy
PROWL_MODEL=qwen3.6-plus              # model id at that endpoint
```

With these set, `bun test` in `packages/pi` exercises the live `modelClient.generate`
path instead of skipping it, and `/prowl search` performs real Qwen synthesis.
